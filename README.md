# 投资研究系统

这是一个面向 GitHub 管理的投资研究工作台。目标不是收集零散资料，而是把书籍、思维模型、行业研究和公司分析沉淀成一套可复用、可验证、可反驳的流程。

本项目当前整合了本地知识包 `多维模型_芒格公司分析知识包/` 的核心方法：芒格多元思维模型、波特竞争框架、格雷厄姆安全边际、财务质量核验、管理层与治理、风险反向清单、三情景估值。

## 分析一家公司

分析任何公司都按 9 步推进。每一步都必须区分事实、推断和假设；涉及最新股价、财报、政策、监管、行业数据时，必须重新核验来源。

1. 公司一句话定义：这家公司本质上是什么生意？
2. 赚钱公式：收入、价格、销量、成本、费用、资本开支和现金流如何形成？
3. 行业结构：需求、供给、竞争、替代、周期和监管如何影响利润？
4. 竞争优势：公司为什么比同行更赚钱，别人为什么抢不走？
5. 财务质量：利润是否是真钱，ROIC 是否高于资本成本，自由现金流是否可靠？
6. 管理层与组织：激励机制、资本配置、分红回购、并购、关联交易是否合理？
7. 风险与反证：什么会毁掉公司，什么指标恶化说明逻辑被破坏？
8. 估值与安全边际：悲观、中性、乐观三种情景下，当前价格是否划算？
9. 最终结论：买入、等待、放弃或继续跟踪，并说明触发条件。

## 提问步骤

每次启动公司研究，先按下面的顺序问。

### 1. 初筛

```text
请帮我快速初筛【公司名称/股票代码】，判断是否值得深入研究。
只输出：一句话定义、赚钱逻辑、可能护城河、最大风险、需要验证的数据、初筛结论。
```

### 2. 行业定位

```text
这家公司属于什么行业类型？请在消费、制造、互联网平台、周期、金融、医药医疗、公用事业中选择最接近的一类，并列出最关键的 5 个行业变量。
```

### 3. 深度公司分析

```text
请使用本项目的公司分析框架、模型库、风险反向清单和估值下注框架，完整分析【公司名称/股票代码】。
不要堆模型，只选择最关键的 5-8 个模型。
输出：结论先行、一句话定义、赚钱公式、行业结构、核心模型与证据、护城河、财务质量、管理层与资本配置、反向清单、估值与安全边际、跟踪指标、最终结论。
如果缺少最新数据，请先列出需要联网核验的信息。
```

### 4. 反向推翻

```text
请不要支持我的观点，而是用反向思维推翻【公司名称/投资逻辑】。
重点回答：我最可能看错哪里？哪个关键假设失效会毁掉投资逻辑？哪些财务、行业、治理或估值指标恶化需要重新评估？
```

### 5. 估值下注

```text
请使用三情景估值方法分析【公司名称】。
输出：当前价格隐含预期、悲观/中性/乐观情景、下行风险、上行空间、胜率赔率、安全边际、机会成本、买入/等待/放弃/继续跟踪结论。
```

### 6. 复盘沉淀

```text
请把这次公司研究沉淀为案例，记录：当时判断、使用模型、支持证据、被忽略的反向信号、最终结果、错误类型、可复用经验。
```

## 输出标准

每份公司研究至少包含：

- 公司卡片：放入 `03_公司分析/公司名/`
- 资料来源台账：记录财报、公告、行业数据、访谈、新闻和估值数据
- 反向清单：明确投资逻辑被破坏的触发条件
- 估值结论：至少包括悲观、中性、乐观三情景
- 跟踪指标：列出观察频率和警戒线
